# Service defaults

How an incoming `Service` gets enriched with category, icon, image URL, and a host-formatted name
before the combiner runs. Pair with [managers.md](managers.md) for where this fits in the write
pipeline.

## The factory

[`ServiceWithDefaultsFactory`](../../src/Dsm.Managers/Services/ServiceWithDefaultsFactory.cs) is the single
public entry point. Called once per incoming service by
[`DashboardCommandProcessor`](../../src/Dsm.Managers/DashboardCommandProcessor.cs), between the
filter/mark-autogenerated step and the per-manager fan-out.

```csharp
public Service CreateWithDefaults(Service service);
```

Pure transformation — returns a new `Service` record; never mutates the input. The factory
handles defaults application and `ImagePath` resolution; icon-prefix resolution and the
fallback chain are now per-manager and live in
[`IconResolver`](../../src/Dsm.Managers/Services/IconResolver.cs) (called from each
`IDashboardManager.WriteServices`).

### Lookup key

Defaults are keyed by service name:

```csharp
var lookupName = service.ServiceDefaultsName ?? service.Name;
_serviceDefaultOptions.Services.TryGetValue(lookupName, out var defaults);
```

`ServiceDefaultsName` is the escape hatch: set it on the provider side (or via the
`<prefix>.service_defaults_name` Docker label) to alias a specific service onto a shared defaults
entry. `Name = "PiKVM HID"` with `ServiceDefaultsName = "PiKVM"` picks up the `pikvm` defaults.

The `Services` dictionary uses `StringComparer.OrdinalIgnoreCase`, so `PiKVM`, `pikvm`, and `PIKVM`
all find the same entry.

When a defaults match is found and the incoming `ServiceDefaultsName` was null, the factory stamps
the resolved lookup key onto the returned service's `ServiceDefaultsName`. This is what lets
downstream consumers — notably the
[Homepage widgets matcher](managers.md#service-widgets) — recover the original defaults key after
`Name` has been rewritten by `NameWithHostnameFormatString`.

### Field precedence

For each field, incoming `Service` wins; the defaults entry fills in only the gaps. Nothing on the
incoming `Service` is ever overwritten.

| Field | Source |
|---|---|
| `Category` | `service.Category` → `defaults.Category` |
| `Icon` | `service.Icon` → `defaults.Icon`. Prefix-driven CDN lookup happens later, per-manager — see [icons.md](icons.md). |
| `ImageUrl` | `service.ImageUrl` → `defaults.ImagePath`, then resolved against `service.Url` |
| `Name` | `defaults.NameWithHostnameFormatString` replaces `%s` with `service.Hostname`; if either is blank, `service.Name` is used verbatim |
| `Hostname`, `Url`, `Ignore`, `Autogenerated` | Copied straight from the input |
| `ServiceDefaultsName` | `service.ServiceDefaultsName` → defaults-lookup key (only when defaults matched and incoming was null) — see [Lookup key](#lookup-key) |

### ImageUrl resolution

`ImagePath` (from defaults or the incoming service's `ImageUrl`) is resolved like so:

- Absolute `http(s)://…` URL → passed through unchanged.
- Relative path + non-empty `service.Url` → joined as a `Uri` against the service URL
  (e.g. `/favicon.png` + `https://host/` → `https://host/favicon.png`).
- Otherwise → passed through as-is (dashboard decides what to do).

## `service-defaults.yaml`

[`service-defaults.yaml`](../../src/service-defaults.yaml) ships with the Manager API and is loaded on
startup as a **required** config source. It defines the `ServiceDefaultOptions` section:

```yaml
ServiceDefaultOptions:
  FallbackIconSourceProviders:
    - HomarrLabs
    - SelfhSt
  Categories:
    media:   { Icon: mdi-multimedia }
    network: { Icon: mdi-network }
  Services:
    plex:
      Category: media
    traefik:
      Category: network
      NameWithHostnameFormatString: "Traefik (%s)"
    grafana:
      Category: monitoring
      ImagePath: /public/img/grafana_icon.svg
```

- `Services` — per-service overlay, keyed by the service's lookup name (see above). Value is a
  [`ServiceConfig`](../../src/Dsm.Managers/Configuration/ServiceConfig.cs):
  `NameWithHostnameFormatString`, `Category`, `Icon`, `ImagePath`.
- `Categories` — used by both Dashy (section icons) and Homepage (`settings.yaml` layout
  icons). Keyed by category name; value is a
  [`CategoryConfig`](../../src/Dsm.Managers/Configuration/CategoryConfig.cs). Each category's
  `Icon` runs through `IconResolver.ResolveIcon` per manager — see
  [icons.md](icons.md#category-icons).
- `FallbackIconSourceProviders` — ordered list of icon sources the resolver probes for
  *services* (not categories) that end up with neither an `Icon` nor an `ImageUrl`. See
  [icons.md](icons.md).

### Overriding from `manager-config.yaml`

Both files bind into the same `ServiceDefaultOptions` section, and `manager-config.yaml` is loaded
**after** `service-defaults.yaml`, so any key it sets wins. The override is leaf-level — whole
dictionary entries are not replaced.

```yaml
# manager-config.yaml — adds Icon to plex without losing the shipped Category
ServiceDefaultOptions:
  Services:
    plex:
      Icon: my-custom-plex-icon
```

Effective `plex` entry: `{ Category: media, Icon: my-custom-plex-icon }` — `Category` survives from
`service-defaults.yaml` because `manager-config.yaml` didn't touch it.

#### List gotcha

List-valued keys are overridden **per-index**, not as a whole list. If `service-defaults.yaml`
declares `FallbackIconSourceProviders: [HomarrLabs, SelfhSt]` and you write
`FallbackIconSourceProviders: [HomarrLabs]` in `manager-config.yaml`, the effective list is still
`[HomarrLabs, SelfhSt]` — your file overrode index 0 (same value) but left index 1 alone. To
shorten the list, set every index you want and leave the rest intentionally empty, or prefer
mutation through environment variables (`DSM_ServiceDefaultOptions__FallbackIconSourceProviders__1=`)
if the deployment layout allows.

This is standard `Microsoft.Extensions.Configuration` behavior, not DSM-specific.

### Key casing

All config is loaded through [`AddNormalizedYamlFile`](../../src/Dsm.Shared/Configuration/NormalizedYamlConfigurationSource.cs),
which strips `_` from keys at load time. So `snake_case`, `camelCase`, and `PascalCase` all bind —
`fallback_icon_source_providers`, `fallbackIconSourceProviders`, and `FallbackIconSourceProviders`
are equivalent. Dashes in keys (e.g. dict keys like `calibre-web` under `Services`) are preserved.

## Icon lookup

`FallbackIconSourceProviders` is consumed by the per-manager
[`IconResolver`](../../src/Dsm.Managers/Services/IconResolver.cs), which is the place where
prefix matching, CDN probing, native pass-through, and the fallback chain all live. The
factory itself doesn't touch icon prefixes anymore — `Icon` flows through verbatim, and the
resolver decides per-manager what gets written to each dashboard's YAML.

Full reference (precedence rules, native-prefix declarations, `MatchedName`, caching, how to
add a new source): [icons.md](icons.md).
