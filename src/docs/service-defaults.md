# Service defaults

How an incoming `Service` gets enriched with category, icon, image URL, and a host-formatted name
before the combiner runs. Pair with [managers.md](managers.md) for where this fits in the write
pipeline.

## The factory

[`ServiceWithDefaultsFactory`](../Dsm.Managers/Services/ServiceWithDefaultsFactory.cs) is the single
public entry point. Called once per incoming service by
[`DashboardCommandProcessor`](../Dsm.Managers/DashboardCommandProcessor.cs), between the
filter/mark-autogenerated step and the per-manager fan-out.

```csharp
public Task<Service> CreateWithDefaultsAsync(Service service);
```

Pure transformation — returns a new `Service` record; never mutates the input.

### Lookup key

Defaults are keyed by service name:

```csharp
var lookupName = service.ServiceDefaultsName ?? service.Name;
_serviceDefaultOptions.Services.TryGetValue(lookupName, out var defaults);
```

`ServiceDefaultsName` is the escape hatch: set it on the provider side (or via the
`<prefix>.service_defaults_name` Docker label) to alias a specific service onto a shared defaults
entry. `Name = "PiKVM HID"` with `ServiceDefaultsName = "PiKVM"` picks up the `pikvm` defaults.

The `Services` dictionary uses `StringComparer.OrdinalIgnoreCase`, so `PiKVM`, `pikvm`, and `PIKVM`
all find the same entry.

### Field precedence

For each field, incoming `Service` wins; the defaults entry fills in only the gaps. Nothing on the
incoming `Service` is ever overwritten.

| Field | Source |
|---|---|
| `Category` | `service.Category` → `defaults.Category` |
| `Icon` | `service.Icon` → `defaults.Icon` (may be replaced by icon-source lookup, see below) |
| `ImageUrl` | `service.ImageUrl` → `defaults.ImagePath`, then resolved against `service.Url` |
| `Name` | `defaults.NameWithHostnameFormatString` replaces `%s` with `service.Hostname`; if either is blank, `service.Name` is used verbatim |
| `Hostname`, `Url`, `Ignore`, `Autogenerated`, `ServiceDefaultsName` | Copied straight from the input |

### ImageUrl resolution

`ImagePath` (from defaults or the incoming service's `ImageUrl`) is resolved like so:

- Absolute `http(s)://…` URL → passed through unchanged.
- Relative path + non-empty `service.Url` → joined as a `Uri` against the service URL
  (e.g. `/favicon.png` + `https://host/` → `https://host/favicon.png`).
- Otherwise → passed through as-is (dashboard decides what to do).

## `service-defaults.yaml`

[`service-defaults.yaml`](../service-defaults.yaml) ships with the Manager API and is loaded on
startup as a **required** config source. It defines the `ServiceDefaultOptions` section:

```yaml
ServiceDefaultOptions:
  FallbackIconSourceProviders:
    - HomarrLabs
    - SelfhSt
  Categories:
    media:   { Icon: fas fa-tv }
    network: { Icon: fas fa-network-wired }
  Services:
    plex:
      Category: media
    traefik:
      Category: network
      NameWithHostnameFormatString: "Traefik (%s)"
    grafana:
      Category: monitoring
      ImagePath: /public/img/grafana_icon.svg
```

- `Services` — per-service overlay, keyed by the service's lookup name (see above). Value is a
  [`ServiceConfig`](../Dsm.Managers/Configuration/ServiceConfig.cs):
  `NameWithHostnameFormatString`, `Category`, `Icon`, `ImagePath`.
- `Categories` — used by dashboard managers (Dashy) to pick a section icon when a category isn't
  otherwise styled. Keyed by category name; value is a
  [`CategoryConfig`](../Dsm.Managers/Configuration/CategoryConfig.cs).
- `FallbackIconSourceProviders` — ordered list of icon sources to probe when a service ends up with
  neither an `Icon` nor an `ImageUrl`. See [icon lookup](#icon-lookup) below.

### Overriding from `manager-config.yaml`

Both files bind into the same `ServiceDefaultOptions` section, and `manager-config.yaml` is loaded
**after** `service-defaults.yaml`, so any key it sets wins. The override is leaf-level — whole
dictionary entries are not replaced.

```yaml
# manager-config.yaml — adds Icon to plex without losing the shipped Category
ServiceDefaultOptions:
  Services:
    plex:
      Icon: my-custom-plex-icon
```

Effective `plex` entry: `{ Category: media, Icon: my-custom-plex-icon }` — `Category` survives from
`service-defaults.yaml` because `manager-config.yaml` didn't touch it.

#### List gotcha

List-valued keys are overridden **per-index**, not as a whole list. If `service-defaults.yaml`
declares `FallbackIconSourceProviders: [HomarrLabs, SelfhSt]` and you write
`FallbackIconSourceProviders: [HomarrLabs]` in `manager-config.yaml`, the effective list is still
`[HomarrLabs, SelfhSt]` — your file overrode index 0 (same value) but left index 1 alone. To
shorten the list, set every index you want and leave the rest intentionally empty, or prefer
mutation through environment variables (`DSM_ServiceDefaultOptions__FallbackIconSourceProviders__1=`)
if the deployment layout allows.

This is standard `Microsoft.Extensions.Configuration` behavior, not DSM-specific.

### Key casing

All config is loaded through [`AddNormalizedYamlFile`](../Dsm.Shared/Configuration/NormalizedYamlConfigurationSource.cs),
which strips `_` from keys at load time. So `snake_case`, `camelCase`, and `PascalCase` all bind —
`fallback_icon_source_providers`, `fallbackIconSourceProviders`, and `FallbackIconSourceProviders`
are equivalent. Dashes in keys (e.g. dict keys like `calibre-web` under `Services`) are preserved.

## Icon lookup

The factory runs two independent icon-resolution steps after pulling defaults.

### 1. Explicit prefix on `Icon`

If `Icon` begins with a registered source's prefix (case-insensitive), the remainder is treated as
an icon name and resolved via that source. On success, the CDN URL lands in `ImageUrl` and `Icon`
is cleared. On a miss (the CDN returns 404 for every probed name variant), the original `Icon`
value is kept and `ImageUrl` stays empty.

| Prefix | Source | CDN |
|---|---|---|
| `hl-` | `HomarrLabsDashboardIconSource` | `https://cdn.jsdelivr.net/gh/homarr-labs/dashboard-icons/png/` |
| `sh-` | `SelfhStDashboardIconSource` | `https://cdn.jsdelivr.net/gh/selfhst/icons/png/` |

```yaml
# Dashy config
items:
  - title: Jellyfin
    url: https://jellyfin.example
    icon: hl-jellyfin         # → https://cdn.jsdelivr.net/.../jellyfin.png on write
```

### 2. Fallback chain

If after defaults are applied the service still has no `Icon` **and** no `ImageUrl`, the factory
iterates `FallbackIconSourceProviders` in order and uses the first source that returns a URL. The
name being probed is the lookup name (`ServiceDefaultsName ?? Name`), so the alias behaves
consistently across every mechanism.

```yaml
ServiceDefaultOptions:
  FallbackIconSourceProviders:
    - HomarrLabs       # tried first
    - SelfhSt          # tried next
```

An empty list disables the fallback entirely.

### Name normalization (both paths)

Source implementations take a logical name and try several variants before giving up:

- lowercase
- spaces stripped (`Home Assistant` → `homeassistant`)
- spaces to hyphens (`Home Assistant` → `home-assistant`)
- dots to hyphens (`Node.js` → `node-js`)

First 200 response wins. Results — hit **and** miss — are cached on the source instance
(singleton DI, so process lifetime). Negative entries expire after 1 hour; positive entries are
kept for the life of the process.

### Adding a new icon source

1. Add a variant to
   [`DashboardIconSourceType`](../Dsm.Managers/Services/IconSources/IDashboardIconSource.cs).
2. If the source is a jsDelivr-backed GitHub repo with the same naming conventions, extend
   [`JsDelivrIconSource`](../Dsm.Managers/Services/IconSources/JsDelivrIconSource.cs) and supply
   `Type`, `Prefix`, `GitHubSlug`, and `HttpClientName`. Otherwise implement `IDashboardIconSource`
   directly.
3. Register in [`HostBuilderConfiguration`](../Dsm.Managers/Hosting/HostBuilderConfiguration.cs):
   ```csharp
   services.AddSingleton<IDashboardIconSource, YourSource>();
   services.AddHttpClient(YourSource.ClientName, c => c.Timeout = TimeSpan.FromSeconds(5));
   ```
4. Opt in — either via `FallbackIconSourceProviders` in `service-defaults.yaml` or
   `manager-config.yaml`, or by having users reference `<prefix>-<name>` directly.
